객체 지향 프로그래밍(OOP) 기초

객체 지향 프로그래밍은 관련된 데이터와 그 데이터를 처리하는 함수(메서드)를 '객체(object)'라는 하나의 단위로 묶어 프로그램을 설계하는 방식입니다. 이 접근법은 현실 세계의 개념을 코드로 모델링하는 것을 단순화하며, 코드의 재사용성을 높이고 확장 및 유지보수가 용이한 시스템을 구축하는 데 도움을 줍니다.

클래스와 객체

  • 클래스(Class): 객체를 만들기 위한 '설계도' 또는 '틀'에 해당합니다.
  • 객체(Object): 클래스의 인스턴스로, 실제로 생성된 실체입니다.

속성과 메서드

  • 클래스 속성(Class Attribute): 클래스 자체에 속한 변수로, 해당 클래스로부터 생성된 모든 객체가 공유합니다.
  • 인스턴스 속성(Instance Attribute): 각 객체마다 고유한 값을 가집니다.
  • 메서드(Method): 클래스 내부에 정의된 함수로, 객체의 동작을 정의합니다.

코드 예시: Dog 클래스

class Dog:
    # 클래스 속성 (모든 Dog 객체가 공유)
    species = "Canis familiaris"

    # 초기화 메서드 (인스턴스 속성 설정)
    def __init__(self, name, age):
        self.name = name  # 인스턴스 속성
        self.age = age    # 인스턴스 속성

    # 인스턴스 메서드
    def speak(self, sound):
        return f"{self.name} says {sound}"
