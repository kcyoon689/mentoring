Shallow Copy vs Deep Copy

Shallow Copy : 최상위 컨테이너만 새로 만들고, 내부 객체는 참조를 공유합니다.

Deep Copy : 모든 하위 객체까지 재귀적으로 복사하여 완전히 독립됩니다.

리스트 중첩 구조 

import copy

original = [[1, 2], [3, 4]]

shallow = copy.copy(original)       # 얕은 복사: 내부 리스트 공유
deep    = copy.deepcopy(original)   # 깊은 복사: 내부까지 새로 복사

original[0][0] = 999

print(f"{original=}")  # [[999, 2], [3, 4]]
print(f"{shallow=}")   # [[999, 2], [3, 4]]  ← 같이 바뀜
print(f"{deep=}")      # [[1, 2], [3, 4]]    ← 그대로

딕셔너리-리스트 혼합 

import copy

cfg = {"model": {"layers": [64, 128], "act": "relu"},
       "train": {"batch": 32, "lr": 1e-3}}

cfg_shallow = copy.copy(cfg)
cfg_deep    = copy.deepcopy(cfg)

cfg["model"]["layers"][0] = 256   # 내부 변경
cfg["train"]["lr"] = 5e-4          # 내부 변경

# 얕은 복사: 내부 딕셔너리 참조 공유 → 값 변함
print(cfg_shallow["model"]["layers"][0])  # 256
print(cfg_shallow["train"]["lr"])          # 0.0005

# 깊은 복사: 완전 독립 → 원래 값 유지
print(cfg_deep["model"]["layers"][0])     # 64
print(cfg_deep["train"]["lr"])             # 0.001

위험 시나리오

  • 중첩 리스트/딕셔너리에서 한쪽 변경이 다른 쪽까지 전파
  • 실험 config/파라미터 스냅샷 오염(재현성 저하)
  • Pandas/NumPy의 뷰(view)와 복사(copy) 혼동으로 원본 훼손

안전 전략

  • 중첩 구조 복사 시 copy.deepcopy() 우선 고려
  • Pandas는 df.copy(deep=True) 명시
  • 복사 직후 id() 또는 얕은 변경 테스트로 참조 분리 확인
