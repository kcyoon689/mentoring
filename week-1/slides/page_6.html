캡슐화와 추상화

캡슐화와 추상화는 코드의 복잡성을 관리하고 안정성을 높이는 데 기여하는 객체 지향 프로그래밍의 핵심 개념입니다.

캡슐화 (Encapsulation)

캡슐화는 데이터(속성)와 해당 데이터를 처리하는 메서드를 하나의 객체로 묶고, 데이터의 일부를 외부로부터 숨기는(정보 은닉) 것을 의미합니다. 이는 객체의 내부 상태가 의도치 않게 변경되는 것을 방지하고, 객체의 사용법을 단순화합니다.

파이썬 접근 제어자

  • Public: 이름 앞에 아무것도 붙이지 않으며, 어디서든 접근 가능합니다. (name)
  • Protected: 이름 앞에 밑줄(_) 하나를 붙이며, 클래스 내부와 상속받은 자식 클래스에서만 사용하는 것을 권장하는 관례입니다. (_breed)
  • Private: 이름 앞에 밑줄(__) 두 개를 붙이며, 클래스 외부에서 직접 접근하는 것을 막습니다. (__age) 
class Person:
    def __init__(self, name, age):
        self.name = name      # Public
        self.__age = age      # Private

    def get_age(self):
        return self.__age

    def set_age(self, age):
        if age > 0:
            self.__age = age

`__age` 속성은 외부에서 직접 접근할 수 없으며, `get_age`와 `set_age` 같은 public 메서드를 통해서만 접근하고 수정할 수 있습니다.

추상화 (Abstraction)

추상화는 객체의 복잡한 내부 구현은 숨기고, 사용자에게는 필수적인 기능만 노출하는 개념입니다. 이를 통해 사용자는 객체의 작동 방식에 대한 모든 세부 사항을 알 필요 없이 객체를 사용할 수 있습니다.

파이썬 추상 클래스

파이썬에서는 추상 클래스(Abstract Class)를 사용하여 추상화를 구현할 수 있습니다. 추상 클래스는 하나 이상의 추상 메서드(구현이 없는 메서드)를 포함하며, 이 클래스를 상속받는 자식 클래스는 반드시 추상 메서드를 구현해야 합니다.

from abc import ABC, abstractmethod

class Vehicle(ABC): # 추상 클래스
    @abstractmethod
    def start(self):
        pass

class Car(Vehicle):
    def start(self):
        print("Starting the car")

# 추상 클래스는 직접 객체를 만들 수 없음
# vehicle = Vehicle() # TypeError 발생

car = Car()
car.start() # 출력: Starting the car

`Vehicle` 클래스는 `start`라는 동작이 필요하다는 '개념'만 정의하고, 실제 구현은 `Car`와 같은 자식 클래스에 위임합니다.

객체 지향 프로그래밍의 이러한 원칙들을 활용하면, 코드를 논리적인 부품(객체)으로 나누어 개발할 수 있습니다.
