검색기 (Retriever)

  • 질문 벡터 변환: 사용자 질문을 고차원 벡터 공간의 임베딩으로 변환
  • 벡터 데이터베이스: 외부 문서를 작은 단위로 분할하고 각 청크를 벡터로 변환
  • 유사도 검색: 질문 벡터와 문서 청크 벡터 간의 유사도 계산 (코사인 유사도)
  • 관련 문서 추출: 유사도 점수가 가장 높은 상위 N개의 문서 청크 추출

DPR(Dense Passage Retrieval) 기술은 질문과 문서의 임베딩을 학습하여 관련성 높은 문서를 효율적으로 검색하도록 최적화된 모델입니다.

생성기 (Generator)

  • 컨텍스트 구성: 검색된 문서 청크와 사용자 질문을 LLM 입력 프롬프트에 추가
  • 답변 생성: 확장된 프롬프트(질문 + 관련 문서)를 기반으로 답변 생성
  • 환각 감소: 외부 지식을 참조함으로써 LLM이 사실과 다른 정보 생성을 줄임

생성기는 주로 대규모 언어 모델(LLM)이 역할을 하며, 제공된 컨텍스트 내의 정보를 활용하여 답변의 정확성과 사실성을 높입니다.

전체 아키텍처

RAG 아키텍처는 사용자 입력부터 최종 답변 생성까지의 데이터 흐름과 각 구성 요소의 상호작용을 통해 LLM이 최신 정보를 반영하고, 특정 도메인 지식을 활용하며, 환각 현상을 줄여 더욱 신뢰할 수 있는 답변을 제공할 수 있도록 합니다.
