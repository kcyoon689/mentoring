프롬프트 엔지니어링: 기본 및 고급 기법

기본 기법

Zero-shot 프롬프팅

모델에게 예시 없이 질문만 제시하여 답변을 얻는 방식

모델의 일반적인 지식을 평가하거나 새로운 문제에 대한 대응 능력을 테스트할 때 주로 활용

예시:

"내일 서울의 오전 9시부터 오후 3시까지의 날씨와 온도를 알려줘."

Few-shot 프롬프팅

몇 가지 예시를 제공하여 LLM이 문제를 이해하고 유사한 방식으로 답변할 수 있도록 돕는 기법

제공되는 예시 수에 따라 1-Shot, 2-Shot 등으로 불리며, 예시 수가 많을수록 정확한 답변을 얻을 가능성이 높아짐

예시:

"입력: 5 * 5, 출력: 25; 입력: 8 + 2, 출력: 10; 입력에 대한 출력을 계산합니다: 7 - 4."

고급 기법

Chain of Thought (CoT) 프롬프팅

모델이 결론에 도달하는 사고 과정을 단계별로 보여주며 문제를 해결하도록 유도하는 방식

복잡한 문제를 해결할 때 각 단계를 명확히 제시하고 논리적인 순서로 문제를 해결하도록 하여, 모델이 문제를 더 깊이 이해하고 논리적 추론을 통해 정교한 답변을 생성하도록 돕습니다

예시:

"주어진 수 중에서 홀수를 모두 더해서 짝수를 만들어 줘: 15, 32, 5, 13, 82, 7, 1. 문제를 단계별로 해결해 봐."

RAG (Retrieval-Augmented Generation)

'검색 증강 생성'의 약자로, LLM이 답변을 생성할 때 외부 데이터베이스에서 관련 정보를 실시간으로 검색하여, 이를 근거로 더 정확하고 신뢰도 높은 답변을 생성하는 기술

LLM의 고질적인 문제인 할루시네이션(환각 현상)을 줄이는 핵심 기술로 주목받고 있습니다

작동 방식:

  1. 데이터셋 벡터화: 외부 데이터를 벡터 데이터베이스에 저장
  2. 질의 입력: 사용자 쿼리를 프롬프트로 받음
  3. 문서 검색: 사용자 쿼리와 유사한 정보 청크를 벡터 데이터베이스에서 검색
  4. 정보 추출: 검색된 문서를 원래 쿼리와 연결하여 LLM에 추가 컨텍스트로 제공
  5. 답변 생성: 연결된 쿼리와 검색된 문서를 포함하는 프롬프트를 LLM에 입력
