객체 인식(Object Recognition): 정의 및 목표

이미지 내 객체의 존재 여부와 종류를 식별하는 컴퓨터 비전 기술

정의

객체 인식은 이미지/비디오 내에서 특정 객체의 존재 여부를 식별하고, 해당 객체가 무엇인지(사람, 자동차, 동물 등) 종류를 분류하는 기술 딥러닝에서 객체 탐지(Object Detection)는 객체 인식의 하위 개념으로, 종류뿐 아니라 위치(Bounding Box)까지 파악

핵심 목표
1

객체의 존재 파악

이미지/비디오 내에서 특정 객체의 존재 여부를 정확히 감지하고 위치를 파악

2

객체의 종류 분류

파악된 객체를 미리 정의된 클래스로 분류 (예: 사람, 자동차, 동물 등)

3

컴퓨터의 시각적 이해

컴퓨터가 사람처럼 이미지를 이해하고 해석할 수 있도록 함.

4

실시간 처리

특히 비디오에서 객체 인식 결과를 실시간으로 제공하는 것이 중요

객체 탐지(Object Detection): 개념과 구조

Recognition에서 한 발 더 — 클래스와 위치(Bounding Box)를 동시에 예측

정의

객체 탐지는 이미지 내 객체의 클래스(종류)와 위치(Bounding Box: x, y, w, h)를 동시에 예측.
인식(Recognition)은 "무엇이 있는가", 탐지(Detection)는 "무엇이 어디에 있는가".

1-Stage 탐지기 — 속도 우선

한 번의 forward pass로 클래스 + 위치를 동시 예측. 실시간 처리에 유리.
대표: YOLO, SSD, RetinaNet

2-Stage 탐지기 — 정밀도 우선

① 후보 영역(Region Proposal) 생성 → ② 각 영역 정밀 분류의 2단계 처리.
대표: R-CNN, Faster R-CNN

객체 탐지 모델 내부를 이해하는 세 가지 핵심 개념

앵커 박스 (Anchor Box)

사전 정의된 레퍼런스 박스
개념

이미지의 각 격자 셀에 다양한 크기·종횡비의 기준 박스를 미리 정의. 모델은 이 기준으로부터의 오프셋(offset)을 학습해 정확한 위치를 예측합니다.

왜 필요한가?

객체의 크기·형태가 다양하므로, 여러 스케일·비율의 앵커를 통해 크고 작은 객체를 모두 탐지할 수 있습니다.

앵커-프리 (최신 추세)

CenterNet, FCOS 등은 앵커 없이 객체 중심점을 직접 예측해 설계를 단순화합니다.

핵심: 다양한 크기·비율의 기준 박스를 사전 정의

IoU (Intersection over Union)

예측 박스 품질 측정 지표
공식
IoU = |예측박스 ∩ 정답박스| / |예측박스 ∪ 정답박스|

0~1 사이 값. 1에 가까울수록 예측이 정확합니다.

훈련 시 활용

IoU ≥ 0.5 → 양성(Positive) 앵커
IoU < 0.3 → 음성(Negative) 앵커
중간(0.3~0.5) → 무시(Ignore)

평가 시 활용

예측 박스 IoU ≥ 임계값 → 정탐(TP)
예측 박스 IoU < 임계값 → 오탐(FP)

핵심: 예측-정답 박스의 겹침 비율 (0~1)

NMS (Non-Maximum Suppression)

중복 박스 제거 후처리
동작 원리

하나의 객체에 여러 박스가 예측될 때, 가장 신뢰도 높은 박스만 남기고 중복을 제거하는 후처리 알고리즘.

처리 단계

① 신뢰도 점수 내림차순 정렬
② 최고 점수 박스 선택(유지)
③ 선택 박스와 IoU ≥ 임계값인 박스 제거
④ 남은 박스에서 ②~③ 반복

Soft-NMS (개선)

제거 대신 IoU에 비례해 점수를 감쇠. 밀집 객체 탐지 성능을 향상합니다.

핵심: 중복 예측 박스 제거 후처리 알고리즘

객체 인식을 위한 대표 딥러닝 모델

R-CNN 계열(2-Stage)과 YOLO 계열(1-Stage)의 발전 흐름

R-CNN 계열  —  2-Stage
R-CNN 기본 모델
이미지당 2,000개의 영역을 제안하고 각 영역에서 특징을 추출해 분류. 계산 비용이 많이 들고 속도가 느림.
Fast R-CNN 속도 개선
R-CNN의 아키텍처를 줄여 처리 시간을 단축하고 정확도를 높인 개선 버전.
Faster R-CNN 실시간
영역 제안 네트워크(RPN)를 도입하여 영역 제안까지 딥러닝으로 처리. 실시간 탐지에 근접.
Mask R-CNN 세그멘테이션
Faster R-CNN을 확장하여 객체 탐지뿐 아니라 픽셀 수준의 인스턴스 세그멘테이션까지 수행.
YOLO 계열 & 기타  —  1-Stage
초기 YOLO 속도 우선
단일 신경망으로 탐지 작업 수행. R-CNN보다 빠르지만 지역화 오류 높고 배경 위양성(FP)은 낮음.
후기 YOLO (v5~) 실시간 표준
속도와 정확도 모두 지속적으로 개선. 현재 실시간 탐지의 사실상 표준 모델.
SSD & RetinaNet 1단계 기타
YOLO와 유사한 간소화 아키텍처. RetinaNet은 Focal Loss로 클래스 불균형 문제 해결.
DETR Transformer
CNN과 트랜스포머를 결합. NMS 없이 Set 예측 방식으로 Faster R-CNN에 필적하는 성능.

이미지 세그멘테이션

픽셀 단위로 이미지를 분할하여 각 픽셀의 클래스를 분류하는 기술

정의

이미지 세그멘테이션은 이미지 내의 각 픽셀을 특정 객체 또는 영역에 따라 분류하는 기술 단순히 객체의 존재 여부나 위치를 파악하는 객체 인식을 넘어, 모든 픽셀에 의미론적 레이블을 할당함으로써 객체의 정확한 형태와 경계를 이해합니다.

이미지 분류

이미지 전체를 하나의 클래스로 분류

위치 정보 없음

객체 탐지

객체의 존재와 위치 (Bounding Box) 파악

대략적인 위치

이미지 세그멘테이션

모든 픽셀에 클래스 레이블 할당

정확한 형태와 경계

주요 목표

  • 각 픽셀에 의미론적 레이블 할당 (도로, 하늘, 사람, 자동차 등)
  • 객체의 정확한 형태와 경계 파악
  • 픽셀 단위 분석으로 이미지 구성 요소를 깊이 있게 이해

이미지 세그멘테이션의 유형

시맨틱 · 인스턴스 · 파놉틱 — 분석 정밀도와 목표에 따라 세 가지로 구분

1

시맨틱 세그멘테이션

Semantic Segmentation
정의

이미지 내 모든 픽셀에 클래스 레이블 할당. 동일 클래스의 객체는 같은 레이블로 표시.

결과물

동일 클래스 → 동일 레이블
(개체 구분 없음)

예시

거리 이미지의 모든 나무를 '나무' 클래스로, 모든 차량을 '차량' 클래스로 분류

평가 지표: mIoU
2

인스턴스 세그멘테이션

Instance Segmentation
정의

시맨틱과 유사하지만, 동일한 클래스 내 개별 객체를 서로 구분하여 분류.

결과물

동일 클래스라도 개체마다
고유한 ID/레이블 부여

예시

거리의 수많은 자동차를 각각 개별 인스턴스로 인식하고 분할

평가 지표: AP (mask IoU)
3

파놉틱 세그멘테이션

Panoptic Segmentation
정의

시맨틱 + 인스턴스의 장점 결합. Thing(개체) + Stuff(배경)을 모두 분류.

결과물

이미지 전체의 포괄적
장면 이해 제공

예시

도로·나무 등 배경은 시맨틱으로, 자동차·보행자는 인스턴스 단위로 분할

평가 지표: PQ = RQ × SQ

객체 인식 vs 이미지 세그멘테이션 비교

구분 객체 인식 (Object Recognition) 이미지 세그멘테이션 (Image Segmentation)

대표 모델 R-CNN, Faster R-CNN, YOLO, SSD FCN, U-Net, DeepLab, Mask R-CNN
주요 활용 객체 감지 및 분류 객체 분할, 배경 제거, 의료 영상 분석
객체 인식: 객체 단위 분석 → 빠르고 가벼움
세그멘테이션: 픽셀 단위 분석 → 정밀하고 상세함

정리

객체 인식

  • 이미지 내 객체의 존재 여부와 종류 식별
  • 객체 탐지: 종류 + 위치 (Bounding Box) 파악
  • 머신러닝 vs 딥러닝 접근 방식
  • 출력: 클래스 레이블 + bbox

대표 탐지 모델

  • R-CNN → Fast R-CNN → Faster R-CNN → Mask R-CNN
  • YOLO 계열: 단일 신경망, 실시간 탐지
  • SSD, RetinaNet: 1단계 탐지기
  • DETR: CNN + Transformer 결합

이미지 세그멘테이션

  • 모든 픽셀에 의미론적 레이블을 할당
  • 시맨틱: 클래스 단위 분류
  • 인스턴스: 개체 단위 구분
  • 파놉틱: 시맨틱 + 인스턴스 통합

두 기술의 핵심 차이

  • 객체 인식: 객체 단위 분석 → 빠르고 효율적
  • 세그멘테이션: 픽셀 단위 분석 → 정밀하고 상세
  • 복잡도: 인식 < 세그멘테이션
  • 활용: 목적에 따라 최적 기술 선택
