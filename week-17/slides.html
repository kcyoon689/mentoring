Overfitting & Regularization Vision · NLP · LLM
01 — 개념

Overfitting이란?

모델 복잡도와 일반화 성능 사이의 균형 문제

🔥

Overfitting — 과적합

훈련 데이터의 노이즈까지 학습해 Train 정확도는 높지만, 새로운 데이터에서 성능이 급락. 모델이 "외워버린" 상태. High Variance.

✅

Good Fit — 적정 학습

훈련 데이터의 패턴을 충분히 학습하면서 새로운 데이터에도 잘 일반화(generalize)되는 이상적 상태.

📉

Underfitting — 과소적합

모델이 너무 단순하거나 학습이 부족해 훈련 데이터조차 제대로 학습 못 함. High Bias.

02 — 데이터 분리

왜 Validation Set이 필요한가?

Train · Valid · Test — 각각의 역할과 분리가 반드시 필요한 이유

TRAIN 70%
VALID 15%
TEST 15%
🏋️ Train Set

모델 파라미터(가중치)를 직접 업데이트하는 데 사용. 모델은 이 데이터를 반복해서 학습하며 Loss를 최소화. 모델이 유일하게 "보는" 데이터.

파라미터 업데이트
🧭 Validation Set

학습 중 하이퍼파라미터 튜닝(lr, 레이어 수, dropout 등)과 Early Stopping 기준으로 사용. 모델 파라미터에는 직접 영향 없음.

Test를 쓰면 안 되는 이유: 매 에폭 Test를 보면 "Test에 맞춘" 결정이 누적 → Test가 오염됨.

하이퍼파라미터 결정
🏁 Test Set

모든 학습·튜닝이 끝난 뒤 단 한 번 평가. 실제 세계 성능의 불편 추정치. 반복 사용하면 점차 "낙관적 편향"이 생겨 신뢰할 수 없게 됨.

최종 성능 추정
01
Leakage 방지
Test를 매번 보면서 결정하면 Test 정보가 모델에 간접 누출. Valid로 격리해야 Test가 진짜 "미래 데이터" 역할을 함.
02
Early Stopping의 기준점
Val Loss가 상승 전환하는 시점을 감지해야 최적 에폭을 결정할 수 있음. Train Loss만 보면 감지 불가.
03
하이퍼파라미터 선택 근거
dropout률, lr, 모델 크기 등 수십 개 조합을 실험할 때 Valid 성능이 유일한 선택 기준. Test는 결론 확인용.

손실 함수에 가중치 페널티 추가. L2(Weight Decay)는 Adam 옵티마이저에서 AdamW로 올바르게 적용.
REGULARIZATION

ALL

ARCHITECTURE

ALL

05 — 정규화 레이어

Batch Norm vs Layer Norm

도메인별로 선택이 갈리는 정규화 레이어의 핵심 차이

📷 Vision / CNN
Batch Normalization

미니배치 축을 따라 각 feature(채널)를 정규화.
CNN의 공간적 특성(H×W)에 잘 맞으며, 이미지 분류·detection 등 Vision 태스크의 표준.

정규화 차원: N(배치), H, W
각 채널 C별로 평균·분산 계산
μ, σ² → 배치 전체에서 추정
✅ 큰 배치에서 안정적, 학습 속도 빠름
❌ 배치 크기 작으면 통계 불안정
❌ RNN/Transformer에는 부적합
❌ 가변 길이 시퀀스 처리 어려움
🤖 NLP / Transformer / LLM
Layer Normalization

각 샘플 내 feature 축을 따라 정규화.
배치 크기에 무관하게 독립적으로 동작. BERT, GPT, LLaMA 등 모든 Transformer 기반 모델의 표준.

정규화 차원: 각 토큰의 Hidden dim
배치·시퀀스 길이와 독립적
μ, σ² → 각 샘플 내에서 계산
✅ 가변 길이 시퀀스에 자연스럽게 적용
✅ 배치 크기 1에서도 동작 (LLM 추론)
✅ RNN·Transformer 전반 호환
❌ Vision CNN에서는 BN보다 성능 낮음

LLM 위치 변형

Pre-LN (GPT-2~): Sublayer 앞에 LN → 학습 안정성 ↑
RMSNorm (LLaMA): 분산만 사용, 계산 효율 ↑

06 — NLP / LLM 전략

NLP · LLM 특화 전략

Transformer 기반 모델 파인튜닝 시 과적합을 다루는 방법들

01
NLP / LLM
Label Smoothing
정답 레이블을 1.0 대신 0.9, 나머지 클래스에 소량(ε=0.1) 분배. 모델이 지나치게 확신하는 것을 방지. 번역·분류에서 효과적.
REGULARIZATION
02
NLP / LLM
LoRA / PEFT
LLM 파인튜닝 시 일부 저랭크 행렬만 학습. 업데이트 파라미터를 극소화해 데이터 부족 상황에서도 과적합 방지. QLoRA로 메모리도 절약.
ARCHITECTURE
03
NLP / LLM
Gradient Clipping
그래디언트 norm에 상한선(max_norm=1.0) 설정. Transformer 학습 시 폭발적 그래디언트로 인한 불안정 및 과적합을 예방.
REGULARIZATION
04
NLP / LLM
Learning Rate Warmup + Decay
초기 웜업 후 cosine/linear 스케줄로 감쇠. Transformer에서 초반부 과적합을 막고 후반부에 세밀하게 수렴하도록 유도.
REGULARIZATION
05
NLP / LLM
AdamW (Weight Decay)
Adam의 L2 정규화 버그를 수정한 옵티마이저. Embedding·LayerNorm 파라미터는 decay 제외. BERT, GPT 등 모든 LLM 학습의 표준.
REGULARIZATION
06
NLP / LLM
Prompt / Data 다양성
Instruction Tuning 시 다양한 템플릿·포맷으로 동일 데이터 증강. 특정 프롬프트 패턴에 과적합되는 것을 방지 (Prompt Overfitting).

07 — NLP / LLM 지표

과적합 진단 지표 — NLP · LLM

Loss 외에 NLP·LLM 도메인에서 과적합을 진단하는 핵심 지표들

Perplexity (PPL)
언어 모델 불확실성 지표
모델이 텍스트를 얼마나 "예측하기 어려워하는지"를 나타냄. Train PPL은 낮은데 Val/Test PPL이 급격히 높으면 과적합 신호. GPT 계열의 핵심 지표.
PPL = exp(H(p,q)) = exp(-1/N · Σ log P(xᵢ))
낮을수록 좋음 (GPT-2: ~35, LLaMA-3: ~6~8)
Train PPL ≪ Val PPL → 과적합
BLEU / ROUGE
생성 품질 평가 지표
BLEU는 번역, ROUGE는 요약의 n-gram 기반 평가. 훈련셋 점수는 높은데 검증셋 BLEU/ROUGE가 낮으면 훈련 문장을 암기한 것. 다양성 지표(distinct-n)도 함께 확인.
BLEU = BP · exp(Σ wₙ · log pₙ)
ROUGE-N = |Ngram 교집합| / |Ngram 참조|
Train BLEU ↑ & Val BLEU ↓ → 과적합
Accuracy / F1 (분류)
NLP 분류 태스크 지표
감성 분석·NLI·분류 파인튜닝 시 핵심. Train F1 ≈ 1.0, Val F1 낮음 → 전형적 과적합. 클래스 불균형 시 Macro F1 사용 권장.
F1 = 2 · (Precision · Recall) / (P + R)
Macro F1 = avg(F1 per class)
Gap = Train F1 − Val F1 > 0.05 → 주의
Memorization Rate
LLM 암기 측정 지표
LLM이 훈련 데이터를 그대로 복사 출력하는 정도. Extractable Memorization이 높으면 과적합 및 개인정보 유출 위험. membership inference attack으로 측정.
Memorized if: P_model(x) > threshold
↔ 훈련 데이터에서 exact 재현 가능
LLM fine-tuning 시 반드시 체크
08 — 요약

도메인별 핵심 체크리스트

상황에 맞는 전략 선택 가이드

①

Train / Valid / Test 반드시 분리

Valid는 하이퍼파라미터 결정과 Early Stopping의 기준. Test는 완성 후 단 한 번 사용. 섞거나 반복 참조하면 성능이 낙관적으로 부풀려짐.

②

진단 먼저 — Loss 곡선 + 도메인 지표

Train/Val Loss 모니터링은 필수. NLP라면 PPL·F1·BLEU도 함께 추적. Gap이 벌어지면 즉시 대응.

③

정규화 레이어는 도메인에 맞게

Vision/CNN → Batch Normalization  |  Transformer/LLM → Layer Normalization (Pre-LN / RMSNorm)

④

LLM 파인튜닝 = Early Stopping + AdamW + LoRA

데이터가 적은 fine-tuning 상황에서는 LoRA로 파라미터를 제한하고, AdamW + warmup 스케줄러를 기본으로 설정.

⑤

하나씩, 검증하면서 쌓기

여러 기법을 동시에 적용하지 말 것. 기법 추가 → Val 지표 확인 → 개선 여부 판단의 순서로 진행.
